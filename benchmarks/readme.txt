SpacerScope benchmark materials

This directory provides benchmark-related materials associated with the SpacerScope manuscript.

Because the complete benchmark archive contains large raw inputs, retained outputs, profiling records, and intermediate analysis files, the full archive has been deposited in Zenodo:

DOI: 10.5281/zenodo.21188872

The GitHub benchmarks directory provides a lightweight local test dataset, run commands, and expected output files that allow users to check whether a downloaded SpacerScope build works correctly on their own computer. The full benchmark archive on Zenodo should be used for reproducing or inspecting the complete manuscript benchmark materials.

Purpose of this GitHub benchmarks directory

This directory is intended to provide:

1. a small simulated test dataset for local functionality checking;
2. the corresponding command used to run SpacerScope on this small dataset;
3. the expected output file generated from the small test;
4. links and guidance for accessing the complete benchmark archive deposited in Zenodo.

The small test dataset is not intended to replace the full benchmark archive. It is provided as a quick verification example to confirm that SpacerScope can run successfully and generate the expected output format.

Full benchmark archive

The full benchmark archive deposited in Zenodo includes the benchmark materials used to support the SpacerScope manuscript, including:

- raw benchmark inputs;
- retained output files;
- run commands for SpacerScope and comparator tools;
- analysis and normalization scripts;
- profiling summaries;
- process-tree runtime and memory records;
- related benchmark records;
- FILE_MANIFEST.xlsx files describing retained benchmark contents.

The full archive is organized by analysis scenario, including:

- Benchmarking via Simulated Genomic Datasets;
- Sensitivity Benchmarking Against CIRCLE-seq Experimental Data;
- Validation via Empirically Verified Off-Target Sites in Oryza sativa;
- Computational Scaling and Performance Benchmarking;
- Workflow-level demonstrations using web-based design platforms;
- Workflow-level Re-analysis in a Polyploid Genome;
- Benchmarking via GUIDE-seq data.

Users who want to inspect the complete benchmark inputs, retained outputs, command records, and profiling summaries should download the Zenodo archive.

Quick local functionality check

A small simulated dataset and the corresponding expected output are provided in this GitHub benchmarks directory to allow users to check whether SpacerScope runs correctly.

The local test materials include:

- a small simulated reference/input dataset;
- the command used to run SpacerScope;
- the expected output file generated by SpacerScope.

To perform the check, users can run the provided command on the small test dataset and compare the generated output with the expected output file included in this directory.

This local test is intended only as a software functionality check. It does not reproduce the full benchmark analyses reported in the manuscript.

Main profiling script

The main script used in the manuscript benchmark workflow for runtime and memory profiling is:

trace_tree.py

This script runs benchmark commands while tracing the process tree and collecting performance information, including elapsed time and memory usage. In the manuscript benchmarks, peak memory was reported as process-level peak proportional set size in GiB.

Notes on benchmark reproducibility

- The GitHub benchmarks directory provides a lightweight test case for checking software functionality.
- The complete benchmark archive is deposited in Zenodo under DOI 10.5281/zenodo.21188872.
- File and folder names in the full archive are preserved from the retained benchmark workflow.
- Some command-note files in the full archive may be written in Chinese because they were preserved from the original local benchmark records.
- Comparator tools may use tool-specific output formats and tool-specific normalization scripts.
- For large public reference genomes, the archive may provide input descriptions, assembly names, command records, retained intermediate files, or processing notes rather than redistributing every complete genome assembly directly.
- The benchmark materials are intended to accompany the manuscript and support reproducibility, inspection, and reuse of the reported analyses.
